Hand Gesture Scoreboard

On-camera gesture detection driving a live dual scoreboard — one .eap, no companion app, no MQTT required.

⬇ Download .eap GitHub →

How it works

The camera runs a TFLite hand gesture model on every frame using the on-chip DLPU. When a gesture is recognised inside a configured zone, its scoring rule fires — and both scores are pushed to a CamOverlay Custom Graphics service in real time.

🤙

Gesture detection

TFLite model runs on-camera at inference time <10 ms using the DLPU — no cloud, no external server.

🎯

Dual zones

Draw Home and Visitors zones over a camera snapshot. Any gesture inside a zone scores for that side.

📺

CamOverlay output

Scores are written directly to a CamOverlay Custom Graphics service — on-screen scoreboard updates instantly.

🔒

Cooldown guard

Per-zone and cross-zone cooldown prevents accidental multi-scoring from model flicker or hand movement.

The model recognises 20 gestures. Default scoring rules are pre-configured for the most common ones — every rule is editable in the Scoring Rules panel.

Gestures with the same default rule (e.g. palm, stop, stop_inverted → all +1) share a cooldown bucket — only the first one detected per frame fires. Any gesture can be assigned any rule in the UI.

Cooldown behaviour

After a gesture scores, two timers start to prevent accidental double-counting:

Same zone
full cooldown (e.g. 3 000 ms)
Other zone
half cooldown (1 500 ms)

The cross-zone guard prevents a gesture made while moving a hand between zones from accidentally scoring on both sides. After the half-cooldown elapses, the other zone is fully open.

Settings & Views

All configuration is in a single settings page. Sections expand/collapse on demand.

🎯 Scoreboard
always visible

Live Home / Visitors score, zone editor (drag-to-draw over camera snapshot), Reset and Push to Overlay buttons.

Scoring Rules
collapsible

Map each gesture to +N, -N, or reset. Set to 0 to ignore a gesture entirely.

CamOverlay Custom Graphics
collapsible

Service ID, field names, text colours (RRRGGGBBB), show/hide overlay gestures, LED blink on score, and cooldown duration.

Games / Sets
collapsible · optional

Win-tracking layer. When a side reaches the target points and leads by the win-by margin, it wins a game — points reset, game count increments. Game counts pushed to field3 / field4.

Live Detection View
collapsible · lazy-loaded

Live camera stream with gesture bounding boxes overlaid. Contains two sub-tabs: Area of Interest (polygon, limits where detections are processed) and Exclude Areas (suppress detections in specific regions).

Detection Settings
collapsible

Confidence threshold, active gesture labels, prioritise accuracy vs. responsiveness, stabilize transition duration.

SD Card Training Capture
collapsible · optional

Saves full-frame JPEGs and YOLO-format label files to SD card on detection — for building site-specific training datasets. Includes Download Archive and Clear buttons.

MQTT
collapsible · optional

Optional MQTT output for integrations. Topics: detectx/detection/<serial>, detectx/events/<serial>/<label>/true|false, detectx/crops/<serial>.

Detection Export
collapsible · optional

Cropped images of each detected gesture via MQTT or HTTP POST. Configurable border padding and output throttle. Gallery shows the 10 most recent crops.

Model
collapsible

Upload a replacement .tflite model and labels.txt directly from the browser — no SSH or Docker rebuild needed.

About
collapsible

Live dashboard: app name/version/vendor, device model/firmware/serial/CPU load, model status/inference time/DLPU, MQTT broker status.

Build & Install

Requirements

Axis camera with ARTPEC-8 (aarch64), AXIS OS 11+, Docker Desktop, and CamOverlay installed on the camera.

Build

git clone https://github.com/[your-org]/handgesture_scoreboard
cd handgesture_scoreboard
docker build --platform linux/arm64 -t handgesture_scoreboard .
# extract .eap from container

Install

Camera → System → Apps → enable Allow unsigned apps → upload .eap → wait ~20 s for model to load.

CamOverlay setup

Create a Custom Graphics service with at least two text fields (field1 = Home score, field2 = Visitors score). Default service ID is 20. If using Games/Sets, add field3 / field4 for game counts.

Troubleshooting

SymptomFix
Overlay not updatingCheck CamOverlay service ID and field names; check the CamOverlay chip on the Scoreboard panel
